Core capabilities

EVERYTHING YOU NEED
TO STAY UNDETECTED

A zero-dependency Go binary with smart analysis, layered encoding, and a full validation engine to ensure behavioral fidelity.

⚡
01
Smart Auto-Detection
22+ feature detectors automatically analyze your script, score its complexity (0–100), and select the optimal profile, level, and flags — no configuration needed.
🔒
02
Behavioral Fidelity
-validate compares stdout, stderr, and exit code byte-for-byte. -profile safe guarantees identical output. Auto-retry fallback with progressively safer settings.
🔀
03
9 Pipeline Transforms
Chain transforms in order: identifier rename, string encryption (XOR/RC4), string dictionary, number encoding, whitespace jitter, opaque predicates, dead code, and anti-reverse.
🧬
04
Polymorphic Fragmentation
Level 5 uses GZip + XOR + up to 256 fragments with Fisher-Yates shuffle and 3 polymorphic decryption templates. No two outputs are the same.
🛡️
05
30+ Safety Protections
Here-strings, interpolated variables, backtick escapes, splatting, scoped variables, class properties, and reserved names are all automatically protected from corruption.
🌐
06
Cross-Platform
Single Go binary. Runs on Windows (CLI + GUI), Linux, and macOS. Supports PowerShell 5.1 and 7.x. Python GUI with 4 modes for a visual workflow.
Obfuscation depth

5 ENCODING LEVELS

From simple char-join to GZip+XOR+fragmentation with structural polymorphism. Each level wraps the transformed script in a different packer.

01
Char Join
Unicode codepoints via [char]::ConvertFromUtf32, join, scriptblock
02
Base64
UTF-8 → Base64, [Convert]::FromBase64String, decode, execute
03
Base64 + Variable
Intermediate variable for 2-step decode — breaks simple pattern matching
04
GZip + Base64
GZip compress, Base64, MemoryStream, GzipStream, StreamReader
05
GZip + XOR + Fragmentation
GZip, XOR 32-byte key, Base64, 256 fragments, Fisher-Yates shuffle, 3 polymorphic templates + decoy vars
Configuration presets

10 OBFUSCATION PROFILES

From zero-transform to maximum obfuscation. Explicit flags always override profile defaults.

safe
Production
Identical output guaranteed. No transforms. FlowSafe on. Best for production use.
light
Minimal
Minimal, fast, compatible. Tight fragmentation only.
balanced
Default
XOR string encoding, 30% string dict, 10% dead code. Good all-around mix.
heavy
Strong
RC4 encryption, identifier renaming, 50% string dict, loose fragments.
stealth
Evasion
RC4, identifier renaming, random seed, polymorphic templates. Pro fragments.
paranoid
Maximum
Every transform enabled. RC4, 60% dict, 30% dead code, anti-debug, all flags.
redteam
Red Ops
Red team operations. RC4, identifier rename, anti-reverse, pro fragmentation.
blueteam
Audit
Deterministic, reproducible builds for blue team auditing. No anti-debug.
size
Minimal output
No encryption, no dead code. Tight fragmentation only. Smallest output.
dev
Debug
Zero transforms, easy human inspection. For development and debugging.
Quick reference

USAGE EXAMPLES

# Smart mode — engine auto-detects best settings
.\obfusps.exe -i script.ps1 -o out.ps1 -auto

# Safe + validated (production)
.\obfusps.exe -i script.ps1 -o out.ps1 -profile safe -validate

# Maximum obfuscation
.\obfusps.exe -i script.ps1 -o out.ps1 -level 5 -profile paranoid -validate

# Analyze without transforming
.\obfusps.exe -i script.ps1 -recommend
# Red team with anti-reverse
.\obfusps.exe -i payload.ps1 -o out.ps1 -profile redteam -level 5 -anti-reverse

# Module file (.psm1)
.\obfusps.exe -i module.psm1 -o out.psm1 -profile balanced -module-aware -use-ast -validate

# Reproducible CI/CD build (seed)
.\obfusps.exe -i script.ps1 -o out.ps1 -level 5 -seed 42 -q

# Generate 10 polymorphic variants (fuzz)
.\obfusps.exe -i script.ps1 -o out.ps1 -level 5 -profile stealth -fuzz 10

# Layer-based (high-level shortcut)
.\obfusps.exe build -i script.ps1 -layers AST,Flow,Encoding,Runtime -profile stealth -o out.ps1
# Custom pipeline: only specific transforms
.\obfusps.exe -i script.ps1 -o out.ps1 -level 5 -pipeline "iden,strenc,numenc" -iden obf -strenc rc4 -numenc

# Full transform chain
# iden → strenc → stringdict → numenc → fmt → cf-opaque → cf-shuffle → dead
.\obfusps.exe -i script.ps1 -o out.ps1 -level 5 -iden obf -strenc rc4 -stringdict 60 -numenc -fmt jitter -cf-opaque -cf-shuffle -deadcode 30

# Stdin/stdout pipeline
Get-Content script.ps1 | .\obfusps.exe -stdin -stdout -level 5 -q > out.ps1
// Programmatic Go API
import "github.com/benzoXdev/obfusps/pkg/obfusps"

result, err := obfusps.Obfuscate(sourceCode, obfusps.Config{
  Level: 5,
  Profile: "balanced",
  Seed: 42,
})

// Install via Go
go install github.com/benzoXdev/obfusps/cmd/obfusps@latest

Complete reference

ALL CLI FLAGS

Every flag supported by the engine. Explicit flags always override profile defaults.

── Core
-level 1..5Obfuscation level — 1=Char Join → 5=GZip+XOR+Frag
-profile <n>Preset: safe | light | balanced | heavy | stealth | paranoid | redteam | blueteam | size | dev
-layers <L,...>AST, Flow, Encoding, Runtime — high-level shortcuts
-pipeline <t,...>iden, strenc, stringdict, numenc, fmt, cf-opaque, cf-shuffle, dead
── Encoding Transforms
-strenc off|xor|rc4String encryption mode — XOR or RC4 cipher
-strkey <hex>Hex key for -strenc (e.g. a1b2c3d4)
-stringdict 0..100String tokenization percentage (split strings into array)
-numencReplace numbers with arithmetic/bitwise expressions
-iden obf|keepIdentifier morphing — obf=rename vars & functions
-fmt off|jitterWhitespace randomization jitter
── Flow Transforms
-cf-opaqueWrap code in always-true opaque predicates
-cf-shuffleReorder contiguous function groups
-deadcode 0..100Dead-code injection probability
-flow-unsafeDisable FlowSafeMode (redteam/paranoid only)
-anti-reverseInject anti-debug / sandbox detection at runtime
── Level 5 / Fragmentation
-frag profile=<p>tight | medium | loose | pro
-minfrag NMinimum fragment count (default 10)
-maxfrag NMaximum fragment count (default 20, max 256)
-no-integrityDisable SHA-256 integrity check (default=true)
── Validation
-validateCompare original vs obfuscated stdout / stderr / exit code
-auto-retryAuto-retry up to 5× with progressively safer settings
-validate-args "..."Arguments passed to script during validation
-validate-stderr <m>strict | ignore (default=strict)
-validate-timeout NTimeout in seconds (default=30)
── Smart / AST / Output
-autoAuto-detect best profile, level, flags from script analysis
-recommendPrint analysis & recommendations — no transformation
-use-astNative PowerShell AST (requires pwsh)
-context-awareSkip strenc for IEX / Add-Type / ScriptBlock::Create
-module-awareProtect Export-ModuleMember and dot-sourced imports
-seed NRNG seed — 0=random, N=reproducible (CI/CD)
-fuzz NGenerate N polymorphic variants
-reportEmit full obfuscation report
-dry-runAnalyze only — no output file written
-qQuiet mode — no banner printed
Real comparison

OBFUSPS VS PSOBF VS POWERCRYPT VS INVOKE

Detailed functional comparison focused on real engine capabilities: AST depth, multi-level model, profiles, fragmentation, validation, reproducibility, and reporting.

ObfusPS
Full-stack

Go engine + smart analysis + validation pipeline. Includes levels 1-5, 10 profiles, auto-retry, entropy scoring, metrics report, and advanced fragmentation.

psobf
Pipeline-first

Modern pipeline model with deterministic seed and strong speed. Good fragmentation and transforms, but less native validation/metrics automation.

PowerCrypt
AST-heavy

Strong AST-centric architecture and very fast execution. Less complete in profile system, validation automation, and reporting depth.

Invoke-Obfuscation
Legacy

Historic reference framework with broad syntax tricks. Flexible for experimentation, but less aligned with modern structured pipeline engineering.
